## Feature Prioritization Framework

### RICE Scoring Method
**Formula:** RICE Score = (Reach × Impact × Confidence) / Effort

#### Scoring Definitions

**Reach:** Number of users/events per time period
- 10+ = Massive (all users, multiple times)
- 5-10 = High (majority of users)
- 2-5 = Medium (significant subset)
- 0.5-2 = Low (small subset)
- <0.5 = Minimal (edge case)

**Impact:** Estimated impact per user
- 3 = Massive impact (transformative)
- 2 = High impact (major improvement)
- 1 = Medium impact (noticeable improvement)
- 0.5 = Low impact (minor improvement)
- 0.25 = Minimal impact (barely noticeable)

**Confidence:** How certain are we about Reach/Impact estimates?
- 100% = High confidence (strong data)
- 80% = Medium confidence (some data)
- 50% = Low confidence (assumptions)

**Effort:** Person-months required
- Estimate total team effort in person-months

### Priority Definitions

**P0 - Critical (Launch Blockers)**
- Must have for MVP launch
- Core value proposition depends on it
- Without it, product is not viable
- Timeline: Current sprint/phase

**P1 - High (Launch Targets)**
- Should have for strong MVP
- Significantly enhances value proposition
- Users will notice its absence
- Timeline: Next 1-2 sprints

**P2 - Medium (Post-Launch)**
- Nice to have for complete experience
- Rounds out feature set
- Competitive parity features
- Timeline: Next 3-6 months

**P3 - Low (Future Consideration)**
- Could have in future versions
- Edge cases or niche use cases
- Innovation experiments
- Timeline: 6+ months or backlog
